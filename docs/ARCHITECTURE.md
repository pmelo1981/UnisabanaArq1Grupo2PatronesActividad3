# Documentación de Arquitectura

## Capas de la Aplicación

1. Domain Layer - Modelos de negocio
2. Application Layer - Lógica de negocio
3. Infrastructure Layer - Acceso a datos
4. Presentation Layer - API REST

## Patrones SOLID

- Dependency Injection en Program.cs
- Repository Pattern para acceso a datos
- Service Pattern para lógica de negocio
